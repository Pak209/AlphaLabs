## 1. Goal of this session

Make AlphaLab's 24/7 crypto path actually *executable* on Alpaca paper, and expand
it beyond Bitcoin-only. Three coupled problems were fixed and the BTC-only idea
generator was generalized to a small Alpaca-tradeable universe.

**Hard constraint discovered (drives every decision below):** Alpaca crypto is
**long-only** (non-marginable, no shorting), and **SOL/XRP are NOT supported**.
So bearish crypto ideas can never execute, and the tradeable universe must be
restricted to coins Alpaca actually lists. Of the user's 5 original targets
(BTC, SOL, LINK, XRP, HYPE), only **BTC / LINK / HYPE** are tradeable.

## 3. Trading behavior — CHANGED

1. **Crypto shorts** now reject cleanly and never reach sizing (previously a
   confusing late error). No bearish crypto idea was ever executable; this just
   makes the rejection honest and early.
2. **Crypto price sizing** can now actually fetch a live crypto quote (was always
   `None` → broke any crypto path needing a price).
3. **Weekend crypto automation now covers BTC + LINK + HYPE**, not just BTC. Each
   coin gets its own signal, deduped against recent crypto theses. This is the
   automated source of crypto ideas (`poll_weekend_crypto`, `sat,sun` every 30m).
4. **Universe change:** SOL and BNB removed from approved crypto tickers (Alpaca
   would have rejected them with a misleading error); HYPE added.

Pattern B/C interlock is unchanged: automated paper orders still require BOTH
`ALPHALAB_SCHEDULER_MODE=paper` AND `ALPHALAB_ALLOW_AUTOMATION_PAPER_TRADES=true`.

## 5. Known caveat / open decision

- **CoinGecko rate limits.** One `poll_weekend_crypto` run now makes **6 uncached
  GETs** (markets + chart × 3 coins). On the free tier the later coins can 429 and
  be skipped that cycle (they're picked up next run; `_safe_market_payload` + the
  90s last-good `_HTTP_CACHE` already degrade gracefully). If this proves flaky in
  production, options: stagger the per-coin fetches, lengthen the cache TTL, or add
  a CoinGecko API key. **Decision pending.**
- **Frontend still BTC-only.** The dashboard's after-hours panel + "generate"
  button (`/api/after-hours/btc`, `app.js`) still target BTC. `generate_after_hours_crypto_ideas()`
  exists in the service but is **not yet wired to an API route or UI**. The
  automated weekend poll already covers all 3 coins, so this is a UI gap, not a
  data gap. **Decision pending:** add a multi-coin route/UI or leave generation to
  the scheduler.
- **ETH/DOGE** remain in `approved_tickers` (Alpaca-supported) but are **not** in
  the `CRYPTO_COINS` generation registry — they'd be accepted if a signal arrived
  from elsewhere, but nothing generates ideas for them. Intentional (universe =
  BTC/LINK/HYPE); revisit if you want ETH/DOGE ideas too.
