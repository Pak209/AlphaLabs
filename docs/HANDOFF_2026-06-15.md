## 3. Current architecture

```
Catalyst → Price/Volume Confirmation → Options/Dark Pool (modifiers) → Risk Engine → Paper Trade
```

| Component | Role |
| --------- | ---- |
| `alpha_lab/live_sources.py` | Catalyst/news ingestion (SEC, Polygon, Benzinga, Tiingo, Newsfilter) + `fetch_polygon_intraday` price/volume snapshot |
| `alpha_lab/service.py` | Core orchestration: scoring composite, hard gate, idea lifecycle, approvals, paper/dry-run execution gating |
| `alpha_lab/analyst.py` | LLM analyst layer (Anthropic primary, OpenAI fallback) for chat + trade explanations; advisory only, no execution authority |
| `alpha_lab/api.py` | FastAPI app; all HTTP routes; opt-in bearer-token middleware on non-GET requests; serves `/`, static, `/sw.js` |
| `alpha_lab/scheduler.py` | Market-hours + weekend cron jobs; Pattern B (dry-run) / C (paper) switch |
| `alpha_lab/repository.py` / `database.py` | SQLite persistence (ideas, trades, signal metrics, audit) |
| `alpha_lab/options_flow.py` / `dark_pool.py` | Conviction-modifier scorers (stub providers today) |
| `paper_trader/` | Execution package (Alpaca paper client + simulated broker); **no `alpha_lab` imports** |
| `alpha_lab/static/` | Vanilla-JS PWA dashboard (no build step): `index.html`, `app.js`, `styles.css`, `sw.js`, manifest, icons |
| `alpha_lab/mcp_server.py` | Pure-stdlib stdio JSON-RPC MCP connector (read + dry-run-only write tools) |

- **Runtime:** Python 3.9, stdlib-preferred (`urllib` for HTTP), FastAPI + uvicorn
  on `127.0.0.1:8787` (prod). Two decoupled packages: `alpha_lab/` and
  `paper_trader/`.
- **Default posture:** paper / dry-run only; the Alpaca base URL is hard-asserted
  to `https://paper-api.alpaca.markets` (live endpoints refused).

## 5. Dedicated server setup status

- **Live.** Dedicated Mac server (Intel macOS; SSH user/host configured in
  `scripts/server.conf`), project at `~/AlphaLab`. Dashboard API on
  `127.0.0.1:8787`.
- Power-hardened for unattended uptime: no sleep on AC + clamshell
  (`pmset disablesleep 1`), FileVault off + auto-login for self-recovery after
  reboot. A multi-hour outage that drains the battery needs a manual power-on.
- Deploy is a one-way rsync **PUSH** from the dev Mac via
  `scripts/deploy_to_old_mac.sh` (config in `scripts/server.conf`); it excludes
  `.env`, the venv, logs, and the SQLite DB, so server-only state is protected.
- **Alpaca paper API health: OK** (`/api/alpaca/health` → `ok:true`,
  `account_status:ACTIVE`, `trading_blocked:false`, paper endpoint confirmed).
- **Caveat:** the dev Mac's automated environment may be unable to SSH to the
  dedicated server (sandbox/network), so deploys should be run from the user's
  own interactive terminal when needed.

**Pending / not done**
- **v35 metrics changes are not deployed** to the old Mac yet (deploy + dashboard
  kickstart still required; see §9).
- Options Flow and Dark Pool providers are **stubs** (`StubOptionsFlowProvider`,
  `StubDarkPoolProvider`) — they return no-data → neutral. No real options/TRF
  vendor selected yet.
- `LLM_ANALYST_ENABLED` confirmed *set* on the server but its literal value
  (`true` vs other) wasn't re-read; live chat returns real replies, so effectively
  enabled — worth an explicit confirm.

**Risks / assumptions**
- The bearer-token gate is **opt-in**: if `ALPHALAB_API_TOKEN` is unset, all writes
  are open. It is currently set on the server, but reads (incl. potentially
  sensitive dashboard data) are always open by design.
- Token is stored in browser `localStorage` and sent as a bearer header; security
  rests on the Tailscale-only network boundary + TLS, not on token rotation.
- iOS sandboxes Safari vs the Home-Screen PWA separately, so the token must be
  entered once in each context (expected, not a bug).
- Single-device server with no auto-restart on power loss; battery-dependent.
- App is not a git repo at `~/AlphaLab` (no local VCS history there); change
  tracking relies on the deploy rsync + these docs.

## 10. What Codex should inspect next

**Security**
- `require_token_for_writes` middleware in `api.py`: confirm it gates **every**
  mutating route (incl. `/api/chat`, `/api/config`, all `/approval/*`,
  `/paper-trade`, `/import*`), uses constant-time comparison, and can't be bypassed
  via method spoofing or non-standard verbs.
- Reads are unauthenticated: assess whether any GET leaks sensitive data
  (account equity, positions, keys-in-error-messages) given tailnet-only exposure.
- `tailscale serve` exposure surface: confirm nothing beyond the intended app is
  proxied, and that the app never trusts client-supplied host/headers.
- LLM provider calls in `analyst.py`: verify no secret leakage into logs/replies
  and that prompt-grounding can't be coerced into emitting execution instructions
  (it has no execution path, but confirm).

**Reliability**
- `fetch_polygon_intraday` failure modes: timeouts, rate limits, malformed
  snapshots — confirm graceful fallback to neutral without crashing the gate.
- Scheduler (`scheduler.py`) idempotency and overlap: ensure KeepAlive restarts
  don't double-fire jobs or place duplicate paper orders; confirm Pattern B/C
  two-flag interlock holds under all paths in `service.py`.
- Service-worker cache correctness after the `v3→v4` bump: stale-asset eviction,
  network-first behavior, offline shell still opens.
- Single-point-of-failure server: behavior after reboot/battery drain.

**Correctness**
- Hard-gate math in `service.py`: catalyst ≥40 AND price/volume ≥55; renormalized
  weights when components drop; the `trend_confirms`/gap-deadband logic that
  prevents contrary-direction volume from clearing the gate (regression-prone).
- `approveAndPaperTrade` in `app.js`: verify the approve→paper-trade chain honors
  `ALPHALAB_REQUIRE_PAPER_APPROVAL` and risk-engine blocks, and that error toasts
  reflect the real server `reasons`.
- Alpaca paper safety assertion (`assert_paper_base_url`): confirm it cannot be
  configured toward a live endpoint.
- Provider-ordering in `analyst.py`: confirm `LLM_PRIMARY` flip and the
  fallback-to-mock semantics, and that `analyst_mode` is reported accurately.
